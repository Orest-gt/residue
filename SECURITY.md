# Security Policy

## Supported Versions

Residue is a security-conscious inference engine. We recommend always running the latest version.

| Version | Supported          |
| ------- | ------------------ |
| 4.2.x   | :white_check_mark: |
| 4.1.x   | :x:                |
| < 4.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within Project Residue, please send an e-mail to Orestis at orestisgatos@gmail.com. All security vulnerabilities will be promptly addressed.

Please include:
- A description of the vulnerability.
- Steps to reproduce (if possible).
- Potential impact.

**Do not open a public GitHub Issue for security vulnerabilities.** We prefer to coordinate a fix before public disclosure to protect users of the engine.
